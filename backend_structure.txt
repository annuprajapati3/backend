📘 API Documentation

Base URL:

http://localhost:5000/api/auth

🔐 1. Authentication APIs
📝 1.1 Signup
➤ Endpoint
POST /signup

➤ Description

Register a new patient or doctor.

➤ Headers
Content-Type: application/json

🧑 Patient Signup Request Body
{
  "name": "Rahul",
  "email": "rahul@gmail.com",
  "password": "123456",
  "role": "patient"
}

👨‍⚕️ Doctor Signup Request Body
{
  "name": "Dr John",
  "email": "drjohn@gmail.com",
  "password": "123456",
  "role": "doctor"
}

✅ Success Response
{
  "message": "Registered successfully"
}

🔑 1.2 Login
➤ Endpoint
POST /login

➤ Description

Login as patient or doctor.

➤ Headers
Content-Type: application/json

➤ Request Body
{
  "email": "drjohn@gmail.com",
  "password": "123456"
}

✅ Success Response
{
  "message": "Login successful",
  "token": "TOKEN_HERE",
  "role": "doctor"
}

🔒 2. Protected API
👤 2.1 Get Logged-in User Profile
📝 2.1.1 For Patient
➤ Endpoint
GET /patient

➤ Description

Returns details of logged-in patient.

➤ Headers (Required)
Authorization: Bearer TOKEN_HERE

✅ Response Example 
{
    "message": "Welcome Patient Dashboard",
    "role": "patient",
    "name": "Rahul",
    "email": "rahul@gmail.com"
}
📝 2.1.2 For Doctor

➤ Description

➤ Headers (Required)
Authorization: Bearer TOKEN_HERE

✅ Response Example 
{
    "message": "Welcome Doctor Dashboard",
    "role": "doctor",
    "name": "Dr John",

}

3. Image & Report APIs
📤 3.1 Upload Image (Patient)
➤ Endpoint
POST /image/upload
➤ Description

Patient uploads medical image + report → system embeds watermark using ML.

➤ Headers
Authorization: Bearer TOKEN_HERE
Content-Type: multipart/form-data
➤ Body (form-data)
Key	Type	Value
file	File	image.png
report	Text	patient report data
patientName	Text	Rahul
doctorId	Text	Doctor MongoDB ID
✅ Response
{
  "message": "Upload successful",
  "data": {
    "_id": "image_id",
    "patientName": "Rahul",
    "reportText": "patient report data",
    "originalImageUrl": "/uploads/original.png",
    "watermarkedImageUrl": "/uploads/wm.png",
    "createdAt": "2026-04-23T07:13:56.478Z"
  }
}
📄 3.2 Get Images (Doctor Dashboard)
➤ Endpoint
GET /image/list
➤ Description

Doctor fetches all reports uploaded for them.

➤ Headers
Authorization: Bearer TOKEN_HERE
✅ Response
[
  {
    "_id": "image_id",
    "patientName": "Rahul",
    "reportText": "patient report data",
    "originalImageUrl": "/uploads/original.png",
    "watermarkedImageUrl": "/uploads/wm.png",
    "createdAt": "2026-04-23T07:13:56.478Z"
  }
]
🔍 3.3 Verify Image (Doctor)
➤ Endpoint
POST /image/verify
➤ Description

Doctor verifies image authenticity using ML model.

➤ Headers
Authorization: Bearer TOKEN_HERE
Content-Type: application/json
➤ Request Body
{
  "imageId": "IMAGE_ID_HERE"
}
✅ Response
{
  "status": "SAFE",
  "details": {
    "geometric_check": "Safe",
    "integrity_check": "Authentic",
    "psnr": 79.64,
    "mse": 0,
    "ssim": 1
  }
}
